# Frontend Implementation Guide (Breach)

This document outlines the step-by-step tasks required to build the frontend application for the Breach Smart Expense Splitting system, based on the finalized `SYSTEM_DESIGN.md`.

## Phase 1: Project Setup & Foundation
- [ ] Initialize frontend project (Next.js / React for web, or Flutter for mobile).
- [ ] Setup State Management (Zustand / Redux / Provider).
- [ ] Setup API Client (Axios / fetch with pre-configured base URL and interceptors).
- [ ] Set up routing structure (Public, Protected, Auth layouts).
- [ ] Implement UI Component Library / Design System (Tailwind CSS / Material UI / Shadcn).
  - typography, buttons, inputs, modals, cards, toast notifications.

## Phase 2: Authentication Flow
- [ ] **Login/Signup Pages**: UI for Email + Password authentication.
- [ ] **OAuth Integration**: "Continue with Google" button.
- [ ] **Auth Context/Provider**: Manage JWT access token in memory/state and handle auto-refresh logic via interceptors.
- [ ] **Protected Routes**: Wrap authenticated pages to prevent unauthorized access.

## Phase 3: Dashboard & User Profile
- [ ] **Overall Dashboard**:
  - Show aggregate stats: "You owe: ₹X", "You are owed: ₹Y", "Net Balance: ₹Z".
  - List of active groups with quick summaries.
- [ ] **Profile Page**: Update display name, avatar, UPI ID.

## Phase 4: Group Management
- [ ] **Create Group Modal**: Form for Name, Description, Currency, and Cover Image.
- [ ] **Group Details View**:
  - Header with group info and current user's net balance in that group.
  - Tabs: Expenses (Ledger), Balances, Members.
- [ ] **Member Management**: Add member by email/username, generate shareable invite link, view member list.

## Phase 5: Expense Recording (The Core Flow)
- [ ] **Add Expense Button/FAB**: Opens a multi-step modal/sheet.
- [ ] **Step 1: Basic Details**: Amount, Description, Category, Date, Paid By (defaults to current user).
- [ ] **Step 2: Participant Selection**: Select which users from the group are involved.
- [ ] **Step 3: Split Method Selection**:
  - **Equal Split UI**: Automatically divide amount among selected members. Show warning if remainder exists.
  - **Percentage Split UI**: Input fields for percentages. Show running total and validate it equals 100%. Auto-calculate ₹ amounts.
  - **Custom Split UI**: Input fields for exact ₹ amounts. Show running total and validate it equals the total expense.
- [ ] **Submit & Optimistic UI**: Send payload to backend, show loading state, and update the ledger immediately on success.

## Phase 6: Transparent Ledger & Balances
- [ ] **Ledger View (Feed)**: Read-only, reverse chronological list of all expenses and settlements in the group.
  - Expense Card: Title, Amount, "You paid / You owe" summary, timestamp.
  - Tap to view full split details.
- [ ] **Group Balances View**: Visual list of who owes whom and who is owed, based on the backend's real-time calculation.

## Phase 7: Settlement & Payments
- [ ] **"Settle Up" Button**: Prominently displayed when the user owes money.
- [ ] **Optimized Settlement UI**: Display the backend-calculated optimal settlement plan (e.g., "Pay Bob ₹500").
- [ ] **Payment Method Selection**: Provide options: UPI App, Scan QR, Card/Netbanking, Mark as Cash.
- [ ] **Razorpay/Stripe Checkout Integration**: Open gateway modal/sheet when digital payment is selected.
- [ ] **QR Code Display**: Show generated UPI QR for scanning.

## Phase 8: Real-Time & Polish
- [ ] **WebSocket Client Integration** (Socket.io):
  - Connect on login, subscribe to user's groups.
  - Listen for `expense:created`, `expense:updated`, `settlement:completed`.
  - Trigger UI refetches or state updates on incoming events.
- [ ] **Push/In-App Notifications**: Toast notifications for real-time events.
- [ ] **Loading States & Skeletons**: Ensure smooth UX during API calls.
- [ ] **Error Handling**: Graceful error boundaries and user-friendly toast messages.
